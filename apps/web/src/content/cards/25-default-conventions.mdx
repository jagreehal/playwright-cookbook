# Card 25: Default conventions (index)

This card does not add new code; it lists the **default conventions** from the cookbook and points to the card that demonstrates each.

| Convention | Card |
|------------|------|
| Tests describe behavior; they don't know selectors | [12-locators-actions-flows](./12-locators-actions-flows/README.md) |
| Selectors: getByRole/getByLabel first; data-testid for repeated/dynamic UI | [13-scoped-queries](./13-scoped-queries/README.md) |
| Every interaction has a done signal (URL/response/toast/state change) | [15-done-signals](./15-done-signals/README.md) |
| No waitForTimeout in committed code | (policy; see [15-done-signals](./15-done-signals/README.md)) |
| Page/region objects expose assertLoaded() and stay small | [12-locators-actions-flows](./12-locators-actions-flows/README.md), [14-region-objects](./14-region-objects/README.md) |
| Cross-page actions return next page object | [12-locators-actions-flows](./12-locators-actions-flows/README.md) (PersonPage.open) |
| Auth via storageState + one UI login smoke test | [19-auth-storage-state](./19-auth-storage-state/README.md) |
| Test data via API factories; UI setup only when testing that UI | [20-api-seeding-cleanup](./20-api-seeding-cleanup/README.md) |
